導入

試験勉強において時間管理は合否を左右すると言っても過言ではありません。特に「応用情報技術者試験」のように午前試験・午後試験で出題形式が異なる試験では、問題ごとの制限時間を意識した練習が重要です。例えば午前試験は1問あたり約1分40秒、午後試験ではケース問題1問あたり30分程度が目安と言われています（note.com）。本番さながらに時間を計りながら問題演習をすることで、時間配分の感覚を身につけることができます（note.com）。私は日々の学習管理にNotionを活用していたため、このNotion上でタイマーを使って問題演習の時間管理ができないかと考えました。しかし調べたところ、Notion単体では音声アラーム付きタイマー機能やテンプレートとして使える汎用タイマーが見当たりませんでした。そこで発想を転換し、Notionの埋め込み機能を活用して、自作のタイマーを組み込むことに挑戦しました。

背景

まず、NotionにはYouTube動画やカレンダーなど様々な外部コンテンツをページ内に埋め込み表示できる強力な機能があります（biz-notion.northsand.co.jp）。実際、社内会議の議事録ページにオンラインのタイマーを埋め込んで時間管理に役立てるといった活用例も報告されています（note.com）。以下の画像はNotionページに外部のストップウォッチ・タイマーを埋め込んだ事例です（会議のタイムキーパー用途）（note.com）。

私もこの埋め込み機能に着目し、試験勉強専用のカウントダウンタイマーを作成してNotionに組み込むことを思い立ちました。もちろん、既成のサービスを埋め込む選択肢もあります。たとえばApptionというサイトでは、Notionに貼り付けて使える便利なウィジェットが数多く提供されており、カウントダウンタイマーもその一つです（zenn.dev）。しかし、Apptionの利用には登録が必要だったり細かなカスタマイズが発生したりするため、手軽さに欠けると感じました（zenn.dev）。また、他のオンラインタイマーサービス（例：stopwatch.onl.jpやtimer.onl.jp）を埋め込む方法もありますが（note.com）、表示に広告が含まれる場合や自分好みに調整できない点が気がかりでした。

以上の理由から、「ならば自分でシンプルなタイマーを作って埋め込もう」と決意しました。幸い、Notionに埋め込む用の簡易Webページであれば、HTML/CSS/JavaScriptだけで十分実現可能です。そこで当初はAIアシスタントのClaudeを活用してReact製のタイマーアプリを試作し、その後ChatGPTの支援により静的なHTML/CSS/JS版へとリファクタリングするプロセスを進めました。

やってみた（タイマー自作への挑戦）

Reactでタイマーアプリ試作

まずはじめに、Claude AIに協力してもらいながらReactを用いたタイマーアプリの試作を行いました。Reactを選んだのは当時の勢いと「せっかくなら最新のフレームワークで」と思ったからです。しかし実装してみると、1からReactアプリを構築するのは学習コストが高く、シンプルなタイマーに対してかえって扱いが大掛かりになってしまいました。

  • ビルドとホスティングの手間: Reactアプリは開発サーバーで動かしているだけではNotionに埋め込めません。本番用にビルドして静的ファイルを生成し、それをどこかにホスティングする必要があります。単純なタイマーのためにWebpackやBabelを用いたビルド工程を踏むのは少々煩雑でした。
  • オーバースペック: ボタン数個と数字表示だけの簡単なUIにも関わらず、React＋Webpackの環境はファイルサイズやロード時間の面で過剰と言えます。Notionページ内でサクッと起動してほしいタイマーに、これだけのオーバーヘッドをかけるのは効率的ではありません。
  • メンテナンス性: 試験勉強期間中に機能変更や調整をしたい場合、Reactコードを修正して再ビルド…という作業が発生します。勉強の合間にそこまで腰を据えて開発環境を回すのは現実的でなく、もっと気軽にコードをいじりたいと感じました。

以上の反省から、方針転換して純粋なHTML/CSS/JavaScriptだけでタイマーを実装し直すことにしました。幸い、Reactで一度ロジックを組んだことでタイマーの要件自体は明確になっていたため、あとはそれをVanilla JSに書き直す作業です。ここではChatGPT (GPT-4) にリファクタリングの相談をしつつ、コードを一から見直しました。

HTML/CSS/JSへのリファクタリング

React版で培った知見をもとに、今度は静的サイトとしてのタイマー実装に取り組みました。具体的なポイントは次の通りです。

  • ファイル分割: ReactではJSX内にHTMLテンプレートやスタイルを含めていましたが、静的版では index.html, styles.css, script.js のようにファイルを分離しました。HTMLにはタイマーの表示枠とボタン類をシンプルにマークアップし、CSSでデザイン調整、JavaScriptで動作を付与するというオーソドックスな構成です。
  • JavaScriptの調整: Reactの状態管理や仰々しいDOM構築は廃止し、直接DOM要素を取得して操作する形に変更しました。例えば残り時間のカウントダウンには、setIntervalで1秒ごとに残秒数を減算し、document.getElementById()で取得した表示要素のテキストを書き換えるだけで十分です。また、タイマー完了時にはclearIntervalでループを停止し、必要に応じてアラート音やメッセージを出す処理を入れることも検討しました。
  • スタイル再設計: 埋め込み先であるNotionのページになじむよう、デザインはシンプルかつ見やすさを重視しました。背景は透過または白ベース、文字は黒字とし、フォントはNotionの雰囲気に合うものを採用しました（基本はシステム既定フォント）。また、ボタンの配置や大きさも、デスクトップとモバイル双方で操作しやすいようCSSメディアクエリで調整しています。

リファクタリングにあたっては、ChatGPTにも「このReactコードをVanilla JSに書き換えるにはどうすれば？」と問いかけ、DOM操作の具体例やシンプルなコードパターンを提案してもらいました。そのおかげでスムーズに再実装が進み、短時間で軽量なタイマーWebページが完成しました。実際、Vanilla JS版はReactのときと比べてページのロードが格段に速くなり、Notionに組み込んだ際の動作も軽快でした。

完成したHTMLページは以下のような構造になりました（一部抜粋）:

<h1>カウントダウンタイマー</h1>
<div id="display">00:00:00</div>
<label>時 <input type="number" id="hours" value="0" min="0"></label>
<label>分 <input type="number" id="minutes" value="1" min="0"></label>
<label>秒 <input type="number" id="seconds" value="40" min="0"></label>
<button id="start">開始</button>
<button id="pause">一時停止</button>
<button id="reset">リセット</button>
<script src="script.js"></script>

このHTMLでは、デフォルトで1分40秒（=100秒）にセットされるようにvalueを設定しています。ユーザーが値を変更することで任意の時間をセット可能です。script.js内では各ボタンのonclickイベントを設定し、開始ボタンでタイマー起動、一時停止やリセットボタンで適宜clearIntervalや時間リセットを行うよう実装しました。こうしたシンプルな作りであれば、ブラウザさえあれば動作するため準備も容易です。

つまづき

  • Notionにタイマー機能がない: これは最初の動機そのものですが、やはりNotion標準機能でタイマーを実現する術がないことを再確認しました。たとえば、ページに現在時刻や残り時間を表示するようなウィジェットはなく、また音を鳴らすガジェットも提供されていません。自動で時間測定するには外部ツールとの併用が必須でした。
  • React開発のオーバーヘッド: 前述の通り、React版で開発を進めた際にビルドやホスティングの手順が煩雑だった点は大きなハードルでした。特に初学者にとってはCreate React Appで大量のファイルが生成された時点で圧倒されるかもしれません。「シンプルな目的に対して本当にReactが必要か？」と疑問に感じ、方向修正せざるを得ませんでした。
  • ホスティングの知識不足: Webページをインターネット上に公開するにはサーバーが必要ですが、自作タイマーを埋め込むにあたり最適な公開方法を模索しました。幸いGitHub Pagesを使えば、静的サイトを無料で簡単にホスティング可能だと知りました（qiita.com）。しかし、設定の細かい部分（リポジトリをPublicにする必要がある等）で最初つまずきました。無料プランのままPrivateリポジトリでPages公開しようとしてうまくいかず、調べてみると無料ユーザーの場合は公開リポジトリでしかGitHub Pagesが使えないことがわかりました（qiita.com）。この点に気づいてからは、リポジトリの可視性をPublicに切り替えて対処しました。
  • Notionへの埋め込み時の調整: タイマーを埋め込んで動作はしたものの、最初はNotion上での見栄えや使い勝手に課題がありました。例えば埋め込みブロックのデフォルトサイズではタイマーUIが窮屈で、一部要素がはみ出してしまったのです。また、ダークモードのNotionで見た際に背景透過の設定が仇となり文字が読みにくくなるという問題も起きました。さらにスマートフォンのNotionアプリで開いた際、レイアウトが崩れて表示されることも確認しました（notion-kyoukasyo.com）。
  • ブラウザ間の差異: 一部の古いブラウザでは<input type="number">の挙動やFlexboxのCSS対応に差異があり、社用PCのIEで正しく動かないなどの問題も発覚しました（※現在Notionは主要クローム系ブラウザで見ることが多いとはいえ、社内展開するなら注意が必要です）。

乗り越えた課題と工夫

  • 外部サービス不使用の利点: Notionへの埋め込みには自作ページを利用する方針としたため、外部サービス由来の広告表示や利用規約に悩まされることはなくなりました。例えば前述のオンラインタイマーではページ下部に広告が表示されていましたが、自作タイマーでは純粋な自分専用のUIだけが表示されます。その結果、学習中に余計な情報が目に入らず集中力を維持できます。
  • GitHub Pagesでの公開: サーバー知識が乏しくても、GitHub Pagesなら静的ファイルをリポジトリに置くだけでWeb公開できます（qiita.com）。具体的には、GitHub上に「umebositabetai.github.io」という名前のパブリックリポジトリを作成し（ユーザー名.github.ioの形式）、その中に先程完成したHTML/CSS/JSファイル一式をプッシュしました。あとはリポジトリのSettingsからPages設定を開き、デプロイ対象のブランチとディレクトリを選択して保存すれば公開完了です（qiita.com）。無料プランの場合はリポジトリをPublicにしておく必要がある点は前述の通りですが（qiita.com）、一度設定してしまえば以後はgit pushするだけで更新が自動デプロイされるので非常に楽です。
  • Notionへの埋め込み手順確立: GitHub Pagesでタイマーが公開できたら、次はそれをNotionに埋め込みます。埋め込み操作自体はシンプルで、Notionの「/embed」コマンドを使います（biz-notion.northsand.co.jp）。以下に手順をまとめます。
  1. 公開URLのコピー: 先ほどGitHub Pagesで公開されたタイマーのURL（例：https://umebositabetai.github.io/）を取得します。
  2. Notionで埋め込みブロック挿入: Notionの任意のページ上で、新しい行に「/embed」と入力しエンターを押します。すると「埋め込み」ブロックが作成され、URL入力用のテキストボックスが表示されます（biz-notion.northsand.co.jp）。
  3. URL貼り付け: テキストボックスにコピーしたタイマーページのURLを貼り付け、「リンクを埋め込む」をクリックします。
  4. 表示確認とサイズ調整: ページ上にタイマーウィジェットが読み込まれ、カウントダウンタイマーの画面が表示されます。必要に応じて埋め込みブロックのサイズをドラッグで調整し、全ての要素が見切れないようにします。

埋め込み後は、Notion上で通常の他コンテンツと並べて配置・閲覧できるようになります。「開始」ボタンを押せばページ内でタイマーがスタートし、設定した時間のカウントダウンを行います。ポイント: Notionの埋め込みは閲覧者のIPアドレス情報を埋め込み先に送信するなど内部的に Iframely 経由で処理されます（notion.com）。難しい設定は不要ですが、埋め込むサイト側でX-Frame-Optionsヘッダーによる表示制限をしていると読み込みできません（Notion自身のページを外部サイトにiframe表示しようとしてブロックされるのが良い例です（note.com））。今回GitHub Pagesで公開したページにはそのような制限をかけていないため、問題なくNotionに埋め込めました。

完成したタイマーの使用感

最終的に完成したカウントダウンタイマーは、Notion上でスムーズに動作しています。GitHub Pagesで公開しているため、Notionだけでなくブラウザから単独でアクセスして使うことも可能です（公開URL: https://umebositabetai.github.io/）。肝心の使い方ですが、例えば午前試験の過去問演習であれば「1分40秒」にセットして開始ボタンを押し、問題を1問解き終えたらリセットして次の問題へ…という流れになります。午後試験対策では「30分」にセットして模擬試験形式で1問を集中演習するといった使い方ができます。時間制限を意識したトレーニングを簡単に実践できるようになり、学習効率のアップにひと役買っています。 埋め込みウィジェットなので、Notionページ内の好きな場所に配置できる柔軟性も便利です。私は「学習ログ」用のNotionデータベースの一角にこのタイマーを埋め込み、いつでも参照・起動できるようにしています。タイマーの直前には「本日の目標解答数」や「現在の時刻」などをメモしておき、スタートボタンを押したら一気に演習に集中する、といったルーチンが定着しました。

実際に使ってみると、自分で工夫して作ったツールだからこそ愛着も湧き、勉強のモチベーション維持にもつながっています。「残り○秒だから急ごう」「時間内に終わった、ペース配分ばっちりだ」といったフィードバックを逐一得られるので、本番同様の緊張感を持ちながら効率よく問題演習ができています。

応用可能性と今後の展開

  • ポモドーロ・タイマーへの応用: 25分作業＋5分休憩といったポモドーロ・テクニック用タイマーにアレンジすれば、仕事や他の勉強にも役立つでしょう。実際、Notionユーザーの中にはポモドーロタイマーウィジェットを埋め込んでいる方もいるようです（youtube.com）。
  • 複数タイマーのテンプレート化: Notion上に複数のタイマーを埋め込んでおき、用途に応じて使い分けるテンプレートページを作ることも可能です。例えば「午前試験演習用1分40秒タイマー」と「午後試験演習用30分タイマー」を2つ並べて配置し、ボタン一つでそれぞれ開始できるようにすると便利かもしれません。あるいは会議議事録テンプレートにストップウォッチとタイマーを両方仕込んでおき、ボタン操作で表示・非表示を切り替えるといった工夫も考えられます（note.com）。
  • 他の学習ツールの埋め込み: タイマー以外にも、自作したちょっとしたWebツールをNotionに埋め込むアイデアは広がります。暗記カード表示アプリ、小テスト生成ツール、進捗グラフ表示など、Notionのデータベースと組み合わせて使えるウィジェットを作れば学習プラットフォームとしてのNotionがさらにパワーアップします。公式に実装されていない機能でも、自前で補える可能性があるのは魅力です。
  • 公開・共有: 今回GitHub上にコードを公開したことで、同じように応用情報試験の勉強をしている仲間にこのタイマーを共有することも容易になりました。URLを教えてあげれば、その人のNotionに同じように埋め込んで使ってもらうことができます。勉強コミュニティでツールをシェアし合うことで、みんなで効率的に学習を進める助けにもなれば嬉しい限りです。

今後は利用者のフィードバックや自分の感じた課題に応じて、タイマーを継続改良していくつもりです。例えば「プリセット時間をボタンで選択できる機能」や「タイマー終了時に通知を飛ばす連携」などがあるとさらに便利になるでしょう。また、埋め込みウィジェット作りのノウハウも今回蓄積できたので、他のアイデアにも積極的に取り組んでみたいと思います。

まとめ

学習効率向上のためにNotionへ自作タイマーを埋め込んだ体験について、導入から完成までの流れを紹介しました。動機は「応用情報技術者試験の時間配分に慣れるため」というシンプルなものですが、その実現のためにReactでの試作、静的サイトへのリファクタ、GitHub Pagesでの公開、Notionへの組み込みと、一連のプロセスを踏むことになりました。途中、技術的な課題やデザイン上の問題にも直面しましたが、その都度調べたり工夫を凝らしたりすることで乗り越えられました。

特に感じたのは、Notionの拡張性と周辺ツールの手軽さです。Notionの埋め込み機能を使えば、他サービスのコンテンツはもちろん、自分で作ったWebページさえワンタッチで組み込めます（biz-notion.northsand.co.jp）。GitHub Pagesのような無料ホスティングも相まって、個人開発の成果物を即座にNotionワークスペースに活かせるのは非常に有用だと実感しました。「Notionにこんな機能があったらいいのに…」と思ったら、自作して埋め込むという選択肢は思いのほか現実的です。

最後に、本記事の内容が同じようにタイマー活用して勉強効率を上げたい方や、Notionをもっと便利に使い倒したい方の参考になれば幸いです。私自身、今回の取り組みで時間管理スキルの向上とツール開発スキルの両面で得るものが大きかったと感じています。今後もNotionを創意工夫で拡張しつつ、楽しく学習を続けていきたいと思います。
